# SageLab v0 — Deferred Items

Generated by /autoplan on 2026-05-26. These items are out of scope for v0 and should be addressed in v1 or later.

## Security / Auth
- **Per-partner auth and audit trail** — v0 uses a single shared `SAGELAB_TOKEN`. v1 needs per-user tokens, session management, and an audit log of who ran what.
- **gitRepoPath allowlist** — When project setup UI is built, restrict `gitRepoPath` to an allowlist defined in `.env` (`SAGELAB_ALLOWED_PATHS`). Do not allow arbitrary filesystem paths from the browser.

## Runtime / Infrastructure
- **Durable run store** — SQLite or file-based persistence so in-progress runs survive server restart.
- **Worker process isolation** — Claude Code currently runs in-process. v1 should run it in a worker process to isolate crashes and resource usage.
- **Explicit workflow manifest** — v0 auto-detects workflows by folder name. v1 should have a `workflows.json` or similar manifest.
- **Reconnect backoff** — 1-second fixed reconnect is fine for v0 local single-user. Revisit when deploying with slow server restarts.

## UI / UX
- **TEXT_DELTA streaming component** — TEXT_DELTA events are suppressed in v0 (raw Claude streaming conflicts with the structured event card philosophy). Needs a design decision: suppress, collapse, or live text panel.
- **Workflow type selector UI** — v0 hardcodes the first available workflow. v1 needs a picker.
- **CompletedCard → wiki links** — v0 uses copy-to-clipboard for artifact paths. v1 should link directly to the committed wiki pages.
- **Research quality as UI primitive** — confidence scores, source quality indicators on ArtifactCard.

## Product / Strategy
- **Competitive monitoring** — Anthropic Claude Projects/Team overlap (flagged at CEO review; no action until v0 ships and gets user feedback).
- **Wiki-first alternative build order** — validate if partners want deliverables (wiki pages) over triggers (chat UI). Revisit after v0 partner pilots.

## Deployment
- **HTTPS + wss://** — Required for production. v0 local-only allows ws://.
- **Process supervision** — systemd / pm2 for production restarts.
- **Secrets management** — `.env` file is fine for local. Production needs a secrets store.
- **Artifact XSS rules** — Wiki content rendered as HTML needs sanitization before v1 ships to partners.
